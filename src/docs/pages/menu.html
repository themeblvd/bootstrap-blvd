General Guidelines

This article discusses three pre-styled navigation blocks you can use in your projects. Before we get to each one, let's comver some general guidelines.

Background and Link Colors

It's important to note that the first style, fs-menu-bar, does not come with any background or link colors applied; so you must add classe or styling for this. However, both the fs-menu-pills and fs-menu-tabs styles have default colors built into their styles.

Horizontal Structure & Responsiveness

By default, all navigation styles are stacked vertically; however, they can be paired with the menu-horz-* classes to display horizontally in a responsive way that allows them to stack vertically again at the desired viewport breakpoint (see below).

This allows us to decouple the menu styling and the horizontal orientation, to ultimately give better responsive flexibility. In other words, menu blocks are styled with a mobile-first approach, and then you can extend them with the desired menu-horz-* class to have them display horizontally at larger viewports.

Below is a table of classes you can use to present your menus horizontally. In the menu block examples throughout this article, you'll see these classes paired with the menu styles.

.menu-horz Menu will never stack and remain horizontal across all viewports. This is not recommended in most cases, and especially not recommended if your menu includes dropdowns.
.menu-horz-xs Menu will stack vertically when viewport is less than 576px. This will generally include smaller mobile devices.
.menu-horz-sm Menu will stack vertically when viewport is less than 768px. This will generally include slightly larger mobile devices (or those in the landscape orientation) and smaller mobile devices.
.menu-horz-md Menu will stack vertically when viewport is less than 992px. This will generally include tablets (in portrait orientation) and any mobile device smaller than that.
.menu-horz-lg Menu will stack vertically when viewport is less than 1200px. This will generally include smaller desktop computers, and all tablets and mobile devices.

HTML Markup Structure

  1. Menu blocks are structured as unordered lists (i.e. <ul>). Any submenus (or dropdowns) should be unordered lists nested within their parent list items.
  2. The top-level unordered list should have the menu style's class — fs-menu-bar, fs-menu-pills, or fs-menu-tabs.
  3. If you're wanting the menu to display horizontally on larger viewports, remember to also add one of the menu-horz-* classes to the top-level unordered list, as discussed above.
  4. All links should have the class menu-btn. This allows for potential non-link placeholders to recieve the same styling (i.e. <span class="menu-btn">), which can sometimes be helpful with dropdowns.
  5. Optionally, you can designate one of the menu items as currently active, by adding the class active to the list item.

  background: #333;

.fs-menu-bar ul {

}
.fs-menu-bar .menu-btn {
  color: #fff;
}
.fs-menu-bar .menu-btn:hover,
.fs-menu-bar .menu-btn:focus {
  background: #101010;
}
