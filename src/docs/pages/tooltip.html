Basic Usage

When setting up a tooltip, you'll designate a trigger, which the user will hover on to display the tooltip help text. Give your trigger element the class fs-tooltip, and a tooltip help element will automatically be generated.

<a href="#" class="fs-tooltip" title="Tooltip Text">est vel posuere congue</a>

<span class="fs-tooltip" data-help="Tooltip Text">nec aliquet dui</span>

Data Attributes

When configuring any individual tooltip trigger element, you can use the following data attributes to customize the generated tooltip help element.

Data Attribute Type Default Description
data-help String None Text to display in tooltip help element. — If this doesn't exist, the title attribute will be used.
data-help-placement String top Optional. Determines placement of tooltip help element, relative to the original trigger element. This can be top, right, bottom or left.
data-help-color String None Optional. A custom color hex like #000 for the generated tooltip help element. Useful for inserting dynamic user-selected colors from a CMS.
data-help-class String None Optional. A custom CSS class to get added to the generated tooltip help element. Useful if you plan to use a custom color for a group of tooltips, that you're styling with CSS.

Accessiblity

Because these tooltips have built with accessiblity as a crucial factor, here are some things to note.

  • A tooltip help element is attached to its trigger via aria-describedby with a uniquely generated ID, and given the aria role tooltip; this aids screen readers in associating the given trigger and help element.
  • A tooltip help element is displayed when its trigger is either hovered or focused on; this allows users to access the help text either by mouse or keyboard.
  • When a tooltip help element is accessed with focus (i.e. keyboard tabbing), it can be closed by losing focus (i.e. tabbing away) or pressing the ESC key on the keyboard.
  • If an trigger element isn't inherently focusable (like a <span> for example), it will become so; it will also receive a visual underline to denote that it can be interacted with.
  • If a trigger element pulls its help text from the title attribute, the title attribute will be removed; this will ensure screen readers don't read duplicate information. — However, if you've explicitly added both title and data-help attributes that are different, the title attribute will remain, in this case.

Placement

By default tooltip help elements will appear above their triggers. However, using the data-help-placement attribute, you can select whether they appear top, right, bottom or left.

Also note that, while it's not demonstrated here, if a tooltip help element is placed top or bottom and its trigger is close to the edge of the browser window, it will be shifted horizontally to fit.

Custom Colors

The color of the tooltips cooresponds to the secondary color of the branding colors. If you're looking to customize the color of tooltips, below are some different methods.

Sass

If you're using Sass, you can customize the variable $fs-testimonial-bg-color.

$fs-tooltip-bg-color: #000;

Static CSS

Or, if you're working with static CSS overrides, here's a snippet you can start with.

.fs-tooltip-help {
    background-color: #000;
    color: #fff;
}
.fs-tooltip-help.top .tooltip-arrow {
    border-top-color: #000;
}
.fs-tooltip-help.bottom .tooltip-arrow {
    border-bottom-color: #000;
}
.fs-tooltip-help.left .tooltip-arrow {
    border-left-color: #000;
}
.fs-tooltip-help.right .tooltip-arrow {
    border-right-color: #000;
}

CSS Class

If you'd like to style preset classes of tooltip help elements, you can pass those classes in via the data-help-class attribute.

<a href="#" class="fs-tooltip" title="Look! It's a red tooltip." data-help-class="red">Red Tooltip</a>

.fs-tooltip-help.red {
    background-color: #c3272b;
    color: #fff;
}
.fs-tooltip-help.red.top .tooltip-arrow {
    border-top-color: #c3272b;
}

Data Attribute

If you're looking to dynamically color an individual tooltip instance, you can pass a hex value through the data-help-color attribute. — This can be useful if you're using a color only once or it came from a user-selected scenario, like from a CMS.
